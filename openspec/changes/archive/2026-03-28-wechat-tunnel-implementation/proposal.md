# wechat-tunnel 实现提案

## 概述

wechat-tunnel 是一个微信消息互通工具，允许 Web 用户通过浏览器与微信用户进行实时聊天。该工具基于腾讯官方的 iLink 协议（微信 ClawBot 插件），提供纯转发式的消息通道，无需持久化存储。

## 目标

1. **实现微信 iLink 协议客户端**：支持 QR 扫码登录、Long-Polling 消息接收、消息发送
2. **提供 Web Chat Demo 界面**：基于 React + Ant Design 的实时聊天界面
3. **实现实时消息转发**：微信消息 ↔ Web 界面的双向实时通信
4. **遵循项目规范**：严格按照 CLAUDE.md 中的 Java 后端规范和前端规范

## 核心价值

- **官方协议**：使用腾讯官方 iLink 协议，区别于灰色方案
- **纯转发模式**：无数据库依赖，轻量级部署
- **实时通信**：基于 WebSocket 的实时消息推送
- **开箱即用**：简单密码认证，快速上手

## 关键设计决策

### 1. 会话模型
**决策**：一个微信号 ↔ 一个 Web 用户（模式 B）

**理由**：
- 实现简单，适合 MVP
- 类似微信网页版体验
- 无需复杂的会话分配逻辑

### 2. 认证方式
**决策**：简单密码认证（方案 2）

**理由**：
- 快速上手，无需用户系统
- 配置文件设置密码
- 适合个人使用或内网环境

### 3. 消息存储
**决策**：不持久化存储，仅内存缓存（方案 A + 内存缓存）

**理由**：
- 无数据库依赖，轻量级部署
- 内存缓存最近 50 条消息，改善刷新体验
- 服务重启后缓存清空（可接受）

### 4. 实时通信
**决策**：WebSocket

**理由**：
- 双向实时通信
- Spring Boot 原生支持
- 前端易于集成

## 风险与限制

### 技术风险
1. **iLink 协议灰测限制**：需要微信账号有 ClawBot 资格
2. **单设备锁**：一个微信号同时只能有一个 Bot Session
3. **Token 过期**：bot_token 可能过期，需重新扫码

### 使用限制
1. **仅支持私聊**：iLink 协议不支持群聊
2. **仅文本消息**：Phase 1 不支持图片/文件/语音
3. **无历史记录**：刷新页面只能看到最近 50 条消息
4. **单用户模式**：同时只能一个 Web 用户使用
